## Юнит 1. Знакомство с данными

Датасет содержит информацию о клиентах банка:

| Признак | Описание |
|---------|----------|
| `age` | Возраст клиента |
| `job` | Профессия |
| `marital` | Семейное положение |
| `education` | Уровень образования |
| `default` | Наличие кредитного дефолта |
| `balance` | Баланс на счёте |
| `housing` | Наличие ипотечного кредита |
| `loan` | Наличие личного кредита |
| `contact` | Тип контакта с клиентом |
| `day` | День последнего контакта |
| `month` | Месяц последнего контакта |
| `duration` | Продолжительность последнего звонка |
| `campaign` | Количество контактов с клиентом за кампанию |
| `pdays` | Количество дней после последнего контакта |
| `previous` | Количество предыдущих контактов |
| `poutcome` | Результат предыдущей кампании |
| `deposit` | Целевая переменная: подписка на депозит (`yes`/`no`) |

## Юнит 4. Базовые модели классификации

- **Логистическая регрессия** — оценка вероятности подписки на депозит.  
- **Решающие деревья** — классификация клиентов по признакам.  
- Метрики оценки: F1-score, precision, recall, accuracy.

## Юнит 5. Ансамбли моделей и построение прогноза

- **Градиентный бустинг** — комбинация слабых моделей для повышения точности.  
- **Стекинг моделей** — объединение нескольких моделей.  
- Важность признаков оценена через `feature_importances_`.  
- Построены визуализации предсказаний и важности признаков.
